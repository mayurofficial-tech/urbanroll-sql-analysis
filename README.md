# UrbanRoll Bike Sharing SQL Analysis

## Project Overview
This project analyzes bike-sharing ride data using SQL to discover
usage patterns, customer behavior, station imbalance, and growth trends.

## Dataset
- Rides table
- Users table
- Stations table
Total rides analyzed: 15,000+

## Key Insights
- Peak commuting hours at 7 AM and 3 PM
- Subscribers take short daily trips
- Casual users take longer leisure trips
- Station imbalance requires bike rebalancing
- Month-over-month user growth observed

## Tools Used
- Google BigQuery (SQL)
- Data Analysis Techniques
- Business Insight Generation
